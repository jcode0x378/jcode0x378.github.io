TemporalAPI

JS 新時間 API Temporal 和 Date 比一比！#

Temporal 好在哪？#

  • Temporal 解決了原本 Date 不支援切換時區，並且需要手動轉換。
  • 日期格式也對新手不太友善，需要手動去計算常發生人為錯誤。
  • 解決 Date 僅支援公曆的問題。

怎麼用？#

$ npm install @js-temporal/polyfill
import { Temporal} from '@js-temporal/polyfill';
// get current timeconst currentTime = Temporal.Now.instant();
console.log(  "Current date and time", currentTime.toString();)
// create a custom date time
const dateTime = Temporal.PlainDateTime.from({  year:2024,  month:8,  day:31,  hour:8,  minute:10,  second:10,})console.log(dateTime.toString());
// manipulate date an timeconst newDateTime = dateTime.add({months:1});console.log(newDateTime.toString());
// calculate differenceconst date1 = Temporal.PlainDate.from("2024-08-31");const date2 = Temporal.PlainDate.from("2024-09-01");const diff = dt2.since(dt1);console.log(diff.days);
// format date and timeconsole.log(`${date1.year}/${date1.month}/${date1.day}`);
// with time zoneconsole.log("with time zone:",currentTime.toZonedDateTimeISO(new Temporal.TimeZone('Asia/Tokyo')).toString());

時間格式#

  • Temporal 採用的是 ISO 8601 格式
  • ZonedDateTime 可以取得完整的整串時間，可以透過 new 或 from() 產生，更推薦用 from() 產生因為支援溢出處理 constrain or reject。
Temporal.ZonedDateTime.from(  { timeZone: 'Asia/Shanghai', year: 2022, month: 2, day: 31 },  { overflow: 'constrain' },).toString();// 溢出報錯Temporal.ZonedDateTime.from(  { timeZone: 'Asia/Shanghai', year: 2022, month: 2, day: 31 },  { overflow: 'reject' },).toString();
  • Instan 會考慮到納秒單位是 bigint， Instan.from 則會帶入時區。
  • PlainDateTime PlainYearMonth PlainMonthDay 用來拿指定時間結構。
  • TimeZone 用來定義時區。
  • Calendar 用來定義日曆系統。
  • Duration 用來表示一段時間。
