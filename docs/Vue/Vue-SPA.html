SPA (single page application)

  • SPA 是用來解決 MPA (multi page application) 的問題，MPA 的缺點是每次切換頁面都需要重新載入，而 SPA 的優點是只有第一次載入時需要重新載入 HTML、CSS、JS，之後切換頁面都不需要重新載入。
  • 當 expression 為 true 時，v-if 會將元素添加到 DOM 中，而 v-show 會將元素的 display 屬性設置為 block，會佔用頁面空間。
  • 當 expression 為 false 時，v-if 會將元素從 DOM 中移除，而 v-show 會將元素的 display 屬性設置為 none，不會佔用頁面空間。

實際差異#

刷新方式#

  • SPA 的優點是只有第一次載入時需要重新載入 HTML、CSS、JS，之後切換頁面都不需要重新載入。
  • MPA 的缺點是每次切換頁面都需要重新載入。

組成#

  • SPA 是由一個主頁面和多個子頁面組成。
  • MPA 的組成是多個主頁面組成。

url 模式#

  • SPA 的 url 模式是 hash 模式，也就是 # 開頭。
  • MPA 的 url 模式是 history 模式，也就是 http:// 開頭。

數據傳遞方式#

  • SPA 的數據傳遞是透過 Vuex 或 Pinia 來傳遞。
  • MPA 的數據傳遞是透過 URL、localStorage、sessionStorage 來傳遞。

優點#

  • 提升使用者體驗
  • 提升開發效率
  • 提升維護效率

缺點#

  • 首頁加載時間較長
  • 對 SEO 不友善
  • 對於網路狀況不佳的環境，可能會有載入問題

DEMO#

class Router {  constructor() {    this.routes = {};    this.currentUrl = '';    this.history = [];    this.currentIndex = -1;

  }
  route(path, callback) {    this.routes[path] = callback;  }
  push(path) {    if (this.routes[path]) {      this.currentUrl = path;      this.history.push(path);      this.currentIndex++;    }  }
